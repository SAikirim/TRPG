Scenario/Ruleset Details
===

Adding a New Scenario
===
1. World Design (World Map First):
   - Create a worldbuilding file in worldbuildings/ (terrain + locations + world_pos)
   - Generate a world map (conditions/methods → see guides/illustration.txt "World Map Generation Rules")
   - Finalize region/terrain placement before starting scenario work
2. Create a {scenario_id}.json file in scenarios/
3. Register metadata in scenarios/index.json (include worldbuilding and ruleset fields)
4. Create entities/{scenario_id}/ directory and write npcs/players/objects/ files
5. Specify scenario_id in game_state.json when starting the game

Adding a New Ruleset
===
1. Create a {ruleset_id}.json file in rulesets/ (refer to fantasy_basic.json)
2. Register it in rulesets/index.json
3. Reference it from the ruleset field in scenario.json

  },

- chapter_themes.bg_type values are used to auto-generate background images (see map_generator.py)

Scenario Start Modes
===
A scenario can be started in two modes.

1. New Game (new_game)
   - Starts with the characters from default_party.
   - Characters with empty names are named by the user during the character creation phase.
   - Loads the initial_state file (scenario-specific game_state initial values).
   - Displays the opening.new_game description and opening.narrative narration.

2. Continue (continue)
   - Carries over characters, inventory, and levels from the prerequisite scenario's save data.
   - Displays the opening.continue description and opening.narrative narration.
   - This mode is unavailable if there is no prerequisite scenario.

prerequisite_required Field
===
- false: Playable as a new game without clearing the prerequisite scenario.
  Presents the user with "New Game" / "Continue" options.
- true: The prerequisite scenario must be cleared before playing.
  Checks the save data for prerequisite scenario completion.
- null or prerequisite itself is null: No prerequisite scenario exists. Only new game is available.

default_party Structure
===
Defines the default party for each scenario. Used when starting a new game.

Each player entry:
{
  "id": 1,
  "name": "",
  "class": "Warrior",
  "level": 1,
  "xp": 0,
  "stats": {"STR": 18, "DEX": 10, "INT": 8, "CON": 14},
  "hp_base": 25,
  "mp_base": 5,
  "starting_inventory": ["Iron Sword", "Leather Shield"],
  "controlled_by": "user"
}

- If name is an empty string, the user is prompted to enter a name during the character creation phase.
- controlled_by: Characters set to "user" are controlled directly by the user. "ai" characters are controlled by the Player Agent.
- Actual hp/mp values are calculated using hp_formula / mp_formula (see character_classes.json).
- starting_inventory: List of items the character starts with at the beginning of the scenario.
- level/xp: Set to match the scenario's recommended level.

Scenario Index (scenarios/index.json)
===
Fields required when registering each scenario:
{
  "id": "scenario_id",
  "title": "Scenario Title",
  "description": "One-line description",
  "genre": "Fantasy",
  "difficulty": "normal",
  "recommended_level": 1,
  "recommended_players": 3,
  "ruleset": "fantasy_basic",
  "worldbuilding": "karendel_world",
  "scenario_file": "scenario.json",
  "initial_state": "game_state_initial.json",
  "available_classes": ["Warrior", "Mage", "Rogue", "Cleric", "Archer"],
  "setting": "Medieval Fantasy Forest Dungeon",
  "estimated_turns": "20~40",
  "tags": ["Dungeon Exploration"],
  "standalone": true,
  "prerequisite": "previous_scenario_id or none",
  "prerequisite_required": false
}

- standalone: If true, the scenario can be played independently as a new game.
- initial_state: Path to the initial game_state file loaded when starting a new game.
- prerequisite / prerequisite_required: See the Scenario Start Modes section.

Game Start Flow (New Game)
===
1. Select a scenario from scenarios/index.json
2. Select a ruleset:
   - Display the ruleset list from rulesets/index.json
   - The scenario's ruleset field is the recommended ruleset (default)
   - If the user does not select one, the recommended ruleset is applied automatically
   - If there is only one ruleset, it is selected automatically
3. Determine the start mode:
   - If no prerequisite exists → New Game
   - If prerequisite exists and prerequisite_required is false → "New Game" / "Continue" selection
   - If prerequisite exists and prerequisite_required is true → Continue only
4. Character Creation (see templates/character_classes.json)
   - New Game: Based on default_party. The user can customize names, classes, and stats.
   - Continue: Characters loaded from the previous save. No additional setup required.
5. Initialize game_state.json with the initial_state file
6. Create entity files in entities/{scenario_id}/
7. Copy active files: scenarios/ → data/scenario.json, rulesets/ → data/rules.json
8. Display opening.narrative + show Chapter 1 map
