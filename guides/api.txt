gm-update Dialogue Record Fields
===
  user_input (string, optional): User's original input. Recorded in events.
  dialogues (array, optional): List of dialogue lines for the turn. Fields for each entry:
    - speaker (string, required): Character name
    - target (string, optional): Conversation partner (if absent, addressed to all / monologue)
    - line (string, required): Actual dialogue text
    - tone (string, optional): Emotion/tone (playful, serious, angry, worried, cold, etc.)
    - action (string, optional): Accompanying action (e.g., snapping fingers)
    - type (string, optional): dialogue / monologue / shout / whisper (default: dialogue)

gm-update Roll Record Fields
===
  Roll record ≠ roll display. show_dice_result only controls narrative display.
  Roll details must always be recorded via one of the two paths below:

  [Path 1] dice_rolls field (when rolling via agent — manual recording)
  dice_rolls (array): Roll details for the turn.
    - type (string, required): skill_check / attack / save / contest
    - roller (string, required): Name of the rolling entity
    - stat (string, required): Ability score used (STR/DEX/INT/CON, etc.)
    - roll (number, required): Dice result value (d20 result)
    - modifier (number, required): Ability score modifier
    - total (number, required): roll + modifier
    - dc (number, required): Target difficulty
    - result (string, required): success / failure / critical / fumble
    - note (string, optional): Additional description (e.g., "pickpocket attempt", "trap detection")

  [Path 2] mechanics field (when game_mechanics.py executes — automatic recording)
  mechanics (array): Roll/heal/combat requests. Results are automatically recorded in events as mechanics_results.
    Example:
    "mechanics": [
      {"type": "skill_check", "player": 1, "stat": "DEX", "dc": 13},
      {"type": "attack", "attacker": 1, "target": 101, "action": "dagger attack"},
      {"type": "heal", "caster": 1, "target": 2},
      {"type": "turn_start"}
    ]
  In this case, dice_rolls does not need to be included (details are included in mechanics_results).

  ⚠️ If a roll occurred but neither dice_rolls nor mechanics is present, the record is missing.
