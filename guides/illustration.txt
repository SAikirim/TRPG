일러스트 시스템 상세
===

이미지 생성 폴백 규칙 (최우선)
===
모든 이미지 생성은 SD WebUI를 우선 사용한다. SD가 동작하지 않으면 Cairo로 폴백한다.

우선순위:
1. SD WebUI API (http://127.0.0.1:7860) — 고품질, GPU 필요
2. Cairo (pycairo) — 벡터 렌더링, CPU만으로 동작, 안티앨리어싱
3. PIL (Pillow) — 최종 폴백, 기본 래스터 렌더링

적용 대상:
| 이미지 종류 | SD 사용 | Cairo 폴백 | PIL 폴백 |
| 배경 일러스트 | O (txt2img) | O (그라디언트+도형) | X |
| 캐릭터 초상화 | O (txt2img + 배경제거) | O (클래스별 실루엣) | X |
| 게임 맵 (미니맵/전체맵) | X | O (타일 렌더링) | X |
| 세계 지도 | O (배경 텍스처) | O (헥스 그리드) | X |
| 오브젝트 | O (txt2img) | O (도형) | X |

SD 동작 확인:
- current_session.json의 sd_illustration 값으로 ON/OFF 제어
- sd_generator.py의 is_sd_enabled()로 확인
- SD WebUI 미응답 시 자동으로 Cairo 폴백 (ConnectionError → _cairo_fallback)

Cairo 사용 원칙:
- 직선보다 곡선(베지어) 우선 사용 — 판타지 느낌
- 안티앨리어싱 항상 활성화 (cairo.ANTIALIAS_BEST)
- 벡터 기반이므로 해상도 자유롭게 설정 가능
- 그라디언트, 투명도, 그림자 적극 활용

SD 시작/종료:
- 게임 세션 시작 시 SD WebUI 자동 확인 (guides/process.txt 참조)
- sd_illustration이 true이면 SD가 동작 중이어야 한다
- SD 미실행 시 Cairo 폴백으로 게임 진행에 영향 없음

SD 일러스트 연동 시스템 (sd_generator.py)
===

SD ON/OFF 토글:
- current_session.json의 sd_illustration 값으로 제어
- 웹 UI의 토글 버튼 또는 /api/illustration/toggle API로 전환 가능

sd_generator.py 동작:
- 비동기 스레드로 SD WebUI API 호출 (request_illustration)
- 모델 자동 확인: dreamshaper_8이 아니면 자동 전환
- 생성 중 상태 추적: _scene_state["generating"]["status"] (idle/generating/error)
- 스레드 안전: threading.Lock으로 scene_state 보호

레이어 시스템
===
비주얼노벨 스타일 레이어 구조:
  [배경 이미지 - 전체]
    +-- [캐릭터 portrait - left/center/right]
    +-- [오브젝트 - 위치 지정]

- 배경 생성 시 기존 레이어 자동 클리어
- remove_layer("이름")으로 특정 레이어 제거
- clear_scene으로 전체 초기화

생성 사이즈:
| 타입                       | 사이즈   |
| 배경 (background/scene)    | 768x512  |
| 초상화 (portrait)          | 384x512  |
| 오브젝트 (object)          | 256x256  |

SD 파라미터
===
- 모델: dreamshaper_8.safetensors
- 샘플러: DPM++ 2M Karras
- 스텝: 20, CFG Scale: 7
- 네거티브: lowres, bad anatomy, bad hands, text, watermark, worst quality, low quality

gm-update 일러스트 payload
===

배경 생성:
  {"illustration": {"type": "background", "prompt": "..."}}

캐릭터 초상화 배치:
  {"illustration": {"type": "portrait", "prompt": "...", "position": "left", "name": "사이키"}}

오브젝트 배치:
  {"illustration": {"type": "object", "prompt": "...", "position": "center-bottom", "name": "보물상자"}}

특정 레이어 제거:
  {"remove_layer": "오크"}

전체 장면 초기화:
  {"clear_scene": true}

Cairo 폴백 (SD OFF 시)
===
- map_generator.py 기반 Cairo(pycairo)/Pillow 코드 생성
- 그라디언트, 베지어, 안티앨리어싱
- 배경: static/illustrations/pixel/ (forest.png, dungeon.png, treasure.png)
- 초상화: static/portraits/pixel/ (player_1.png ~ player_3.png)

세계 지도 생성 규칙
===
generate_world_map() 함수가 세계 지도를 자동 생성한다.

생성 흐름:
1. worldbuilding.json에서 terrain + locations + world_info 로드
2. Cairo로 지형별 그라디언트 색상맵 생성 (가우시안 블러 적용)
3. SD img2img로 판타지 텍스처 변환 시도 (캐시: world_map_bg.webp)
4. SD 실패 시 Cairo 양피지 + 헥스 그리드 폴백
5. Cairo 오버레이: 강(베지어 곡선) + 도로(메인/샛길) + 지역 마커 + 라벨 + 현재 위치
6. 장식: 세계 이름 + 테두리 + 나침반

세계 지도 제목:
- worldbuilding.json의 world_info.name을 읽어 "{이름} 세계 지도"로 표시
- name이 없으면 "세계 지도"로 폴백

자동 생성 트리거:
- Flask 서버 시작 시 (app.py)
- gm-update 호출 시 (세계관 변경 반영)
- 수동: python -c "from core.map_generator import generate_world_map; generate_world_map()"

SD 배경 캐시:
- static/maps/world/{세계관이름}/background_sd.webp에 SD 생성 배경을 세계관별 캐시
- static/maps/world/{세계관이름}/background_cairo.webp에 Cairo 생성 배경 캐시
- 세계관 전환 시 기존 캐시는 보존, 복귀 시 즉시 재사용
- 좌표/지형 변경 시에도 배경 캐시는 보존 (오버레이만 재생성)
- 배경 재생성이 필요한 경우에만 캐시 삭제: rm static/maps/world/{이름}/background_sd.webp
- 좌표 변경 시 캐시를 삭제하지 않는다 — 오버레이만 재생성하면 충분

AnimateDiff
===
- 모션 모듈: extensions/sd-webui-animatediff/model/mm_sd_v15_v2.ckpt (1.7GB)
- WebP 직접 출력 지원: format: ["WEBP"]
- 8GB VRAM에서 8~12프레임 생성 가능

시나리오 사전 생성 이미지
===
시나리오 Agent가 시나리오를 생성하면, 전체 스토리를 기반으로 필요한 모든 이미지를 사전 생성한다.
게임 중 실시간 생성 대기 없이 즉시 표시 가능. 포맷은 WebP.

사전 생성 대상:
| 분류              | 파일명 규칙                    | 크기    | 배경 |
| 챕터 배경         | background_{name}.webp         | 896x512 | 있음 |
| 장소/이벤트 배경  | background_{name}.webp         | 896x512 | 있음 |
| NPC/몬스터        | portrait_{name}.webp           | 384x512 | 투명 |
| 플레이어 캐릭터   | portrait_{name}.webp           | 384x512 | 투명 |
| 오브젝트          | object_{name}.webp             | 256x256 | 투명 |

인물/오브젝트 투명 배경 규칙:
- 인물과 오브젝트는 배경 없이(투명) 생성한다 -> 어떤 배경 위에든 합성 가능
- SD 생성 후 transparent-background 라이브러리로 자동 배경 제거
- negative prompt에 자동으로 "detailed background, scenery, landscape" 추가됨
- 프롬프트에 "simple background, solid color background" 포함 권장

시나리오 이미지 생성 절차:
1. 시나리오 Agent가 scenario.json 생성 시 전체 스토리 파악
2. 필요한 이미지 목록 추출: 모든 챕터 배경, 등장 NPC, 주요 오브젝트, 주요 장소
3. 각 항목에 대한 영문 프롬프트 작성
4. SD API로 순차 생성 (VRAM 제약으로 동시 생성 금지)
5. 품질 확인 -> 문제 시 프롬프트 개선 후 재생성
6. static/illustrations/sd/ 및 static/portraits/sd/에 저장
7. git commit + push (docs/ 자동 동기화)

기본 배경 자동 표시:
- 웹 UI는 현재 챕터에 맞는 배경을 자동 표시한다
- 우선순위: GM 요청 장면 > SD 사전 생성 배경 > Cairo 폴백 배경
- SD OFF 시에도 Cairo 배경이 항상 표시된다 (빈 화면 없음)
- app.py의 /api/illustration 엔드포인트가 default_bg (sd/pixel 경로)를 제공

일러스트 생성 기준
===
- 비중 없는 NPC/생물체(짐말, 배경 동물 등)는 일러스트를 생성하지 않는다
- 유저가 명시적으로 요청한 경우에만 생성
- 일러스트 이미지는 원본 크기를 사용한다 — 미리 축소하지 않음
- 웹에서 CSS로 적절히 표시 (max-width/max-height로 제어)

NPC 초상화 생성 시 필수 사항
===
- SD로 NPC 초상화 생성 시 반드시 entities/{scenario_id}/npcs/npc_{id}.json의 appearance 필드를 참조
- appearance의 age, build, skin, hair, face, outfit, notable을 영문 프롬프트에 반영
- race 필드도 반드시 반영 (인간/엘프/오크 등)
- 세계관 설정(중세 판타지)에 맞는 의상/소품 사용 — 현대풍 금지
- 생성 후 외형이 엔티티 설정과 일치하는지 확인 필수

초상화/오브젝트 자동 프롬프트 생성
===
- request_illustration(type="portrait", prompt="", name="이름") 호출 시 prompt가 비어 있으면 자동 생성
- 오브젝트(type="object")도 동일하게 적용
- 동작 방식:
  1. entities/{scenario_id}/npcs/ 및 players/ 디렉토리에서 name이 일치하는 엔티티 JSON 검색
  2. 찾으면 race + appearance 필드를 영문 SD 프롬프트로 변환 (_appearance_to_prompt)
  3. race는 한국어→영어 매핑 테이블 사용 (인간→human, 엘프→elf, 말→horse 등)
  4. appearance의 age, build, skin, hair, face, outfit, notable을 순서대로 프롬프트에 추가
  5. 스타일 키워드 자동 추가: fantasy, semi-realistic, upper body portrait, simple background, masterpiece, best quality
  6. 엔티티를 찾지 못하면 "fantasy character portrait, {name}, simple background, masterpiece"로 폴백
- prompt가 명시적으로 제공된 경우 자동 생성을 건너뛰고 제공된 prompt를 그대로 사용
- 이 기능으로 빈 프롬프트로 초상화 요청 시에도 캐릭터 설정에 맞는 이미지가 생성된다

캐릭터 외형 일관성 규칙
===
- 동일 인물은 항상 같은 외형(얼굴, 체형, 의상, 머리색)을 유지해야 한다
- 캐릭터별 기준 이미지(reference)를 static/portraits/sd/ 또는 static/portraits/pixel/에 저장
- 새 장면에서 같은 캐릭터가 등장할 때 기존 이미지를 재활용한다
- 외형이 변경되는 경우(장비 변경, 부상 등)에만 새 이미지를 생성
- 동작/포즈만 바꿔야 할 경우: img2img로 기존 이미지를 기반으로 변형
- 레이어 초상화는 반드시 투명 배경 -- 배경이 있는 초상화는 메인 배경을 가림
- SD 생성 시 transparent-background 라이브러리로 자동 배경 제거
- 배경 제거 실패 시 로그 경고 + 그대로 저장 (수동 확인 필요)

캐릭터 이미지 파일 규칙:
| 용도              | 경로             | 이름 규칙                          | 배경      |
| 기준 이미지(정면) | portraits/sd/    | portrait_{name}.webp               | 투명      |
| 동작 변형         | portraits/sd/    | portrait_{name}_{action}.webp      | 투명      |
| 프로필 카드용     | portraits/pixel/ | player_{id}.png                    | 원형 마스크|

예시:
- portrait_할란.webp (기본 -- 정면)
- portrait_할란_injured.webp (부상 상태)
- portrait_미라.webp (기본)
- portrait_미라_angry.webp (화난 표정)

이모지 렌더링 규칙
===
- 맵 PNG 이미지에서 사용하는 이모지는 Windows seguiemj.ttf 호환이어야 한다
- map_generator.py의 safe_emoji() 함수가 자동으로 미지원 이모지를 폴백 처리
- 미지원 판정 기준: 30x30px 렌더링 시 alpha 채널 픽셀 < 150개 (컬러 글리프 없음)
- 새 이모지 추가 시 반드시 safe_emoji()를 거쳐야 한다
- 폴백 매핑: EMOJI_FALLBACK 딕셔너리에 등록
- HTML 웹 UI에서는 브라우저가 렌더링하므로 이 제한 없음 (PNG 맵 전용)
- 오브젝트 엔티티 JSON의 icon 필드는 원본 이모지 유지 가능 — 렌더링 시 safe_emoji가 처리

이미지 품질 관리
===
- 생성 후 품질 확인 필수: 흐릿함(blurry), 초점 이탈(out of focus), 왜곡(deformed) 체크
- 문제 있는 이미지는 프롬프트 개선 후 재생성
- 프롬프트 개선 팁:
  - 흐릿한 결과: negative에 "blurry, out of focus, bokeh, depth of field" 추가
  - 손/신체 왜곡: negative에 "bad hands, bad anatomy, deformed, extra fingers" 추가
  - 초점 안 맞음: prompt에 "sharp focus, 4k" 추가
  - 배경에 인물 포함됨: negative에 "people, characters, person" 추가
